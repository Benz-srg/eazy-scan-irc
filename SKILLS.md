# EazyScan — Manday Estimation Heuristics

ตารางอ้างอิงคร่าว ๆ สำหรับ AI ใช้ตั้งต้นการประเมิน (วันทำงาน, ทีม full-stack 2–3 คน)
ปรับขึ้น/ลงตามความซับซ้อนจริงใน transcript และระบุสมมติฐานเสมอ

| โมดูล / งาน | ช่วงทั่วไป (วัน) | ปัจจัยที่ดันให้สูงขึ้น |
|---|---|---|
| Auth + RBAC (สิทธิ์ผู้ใช้) | 5–9 | social login, SSO, หลายบทบาท |
| จัดการสินค้า (Product CRUD) | 4–7 | SKU/variant, รูปหลายรูป, นำเข้า bulk |
| สต็อก/คลัง (Inventory) | 6–12 | หลายสาขา, real-time sync, จองสต็อก |
| ออเดอร์ PO/SO | 8–14 | เอกสาร, สถานะหลายชั้น, คืนสินค้า |
| CRM | 4–8 | segment, ประวัติ, การตลาด |
| Dashboard & Report | 6–12 | charts หลากหลาย, export, filter ลึก |
| Payment Gateway | 3–7 | หลายผู้ให้บริการ, ใบกำกับภาษี |
| Notification (LINE/Email) | 2–5 | template, webhook, schedule |
| AI / Insight | 5–10 | คุณภาพข้อมูล, การเทรน/ปรับโมเดล |
| Booking / คิว | 5–10 | ปฏิทิน, ทรัพยากร, แจ้งเตือน |
| Infra / DevOps / CI-CD | 5–10 | multi-env, monitoring, scaling |
| QA / ทดสอบ | 10–15% ของงานพัฒนา | ระบบเงิน/สต็อกต้องทดสอบหนัก |

## กฎการรวม

- ผลรวม `mandayMin` ≈ ผลรวม `min` ของทุกโมดูล; `mandayMax` เช่นเดียวกัน
- บวกบัฟเฟอร์ความเสี่ยงเมื่อมีจุดเสี่ยงสูง (เช่น real-time sync หลายช่องทาง)
- งบไม่ระบุ / timeline สั้น → เสนอทำ MVP ก่อน และระบุใน risks

## งานที่ไม่ใช่ซอฟต์แวร์ (research / ตั้งต้นธุรกิจ / ops)

ถ้า transcript เป็นงานทั่วไป ไม่ใช่ dev ให้ลงเป็นเฟส อย่าคืน 0:

| งาน | ช่วงทั่วไป (วัน) | หมายเหตุ |
|---|---|---|
| สำรวจ/หาทำเล (location research) | 2–5 | ลงพื้นที่, เทียบทำเล, ค่าเช่า |
| ศึกษาตลาด/คู่แข่ง (market research) | 3–6 | สำรวจราคา, กลุ่มลูกค้า |
| ศึกษาความเป็นไปได้ (feasibility) | 2–4 | ต้นทุน, จุดคุ้มทุน |
| วางแผนดำเนินงาน (planning) | 1–3 | timeline, ทรัพยากร |
| ทดลอง/นำร่อง (pilot) | 3–7 | ทดสอบจริงก่อน scale |
| จัดหา/ตั้งต้น (setup/procurement) | 2–5 | อุปกรณ์, วัตถุดิบ, ใบอนุญาต |

หลัก: ประเมินจากเป้าหมายที่ลูกค้าพูด + ระบุสมมติฐาน + confidence ต่ำเมื่อข้อมูลน้อย

## ระดับความเสี่ยง (level)

- `สูง` — กระทบสถาปัตยกรรม/ข้อมูล หรือมีโอกาส race condition / data loss
- `กลาง` — ขอบเขตไม่ชัด, dependency ภายนอก, timeline ตึง
- `ต่ำ` — งานมาตรฐานที่ทีมเคยทำ
