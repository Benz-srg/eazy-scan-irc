# EazyScan — Project Context

ระบบผู้ช่วย AI ประเมิน Manday: รับเสียง requirement จากลูกค้า → ถอดเสียง → วิเคราะห์เป็น SOW + ประเมิน Manday + ความเสี่ยง + สมมติฐาน + ข้อมูลที่ขาด + คำถามที่ควรถาม

## บริบทธุรกิจ

- ผู้ใช้คือ Project Manager / Pre-sales ในบริษัทซอฟต์แวร์ไทย
- ลูกค้ามักส่ง requirement ผ่านการพูดคุย/voice memo ไม่เป็นเอกสาร
- เป้าหมาย: ลดเวลา PM ในการฟัง สรุป และตีราคา จากชั่วโมงเหลือไม่กี่นาที
- ผลลัพธ์ต้องพร้อมส่งลูกค้า (DOCX/PDF) และน่าเชื่อถือ (มี evidence + confidence)

## หลักการประเมิน (ใช้เป็นฐานความรู้ของ AI)

- ประเมินเป็น **ช่วง (min–max)** เสมอ ไม่ใช่ตัวเลขเดียว เพื่อสะท้อนความไม่แน่นอน
- แยกประเมิน **รายโมดูล/เฟส** แล้วรวม — ไม่ประเมินเป็นก้อนเดียว
- รวมงานที่มักถูกลืม: **QA/ทดสอบ, DevOps/Infra, การ integrate, เอกสาร** เสมอ
- ระบุ **สมมติฐาน** ทุกครั้งที่ requirement กำกวม แทนการเดาตัวเลข
- ถ้าข้อมูลไม่พอประเมิน → ใส่ใน `missing` + `questions` ไม่ใช่แต่งขึ้น
- `confidence` ต่ำเมื่อ requirement สั้น/กำกวม/ไม่ระบุงบ-ปริมาณ-integration
- **ประเมินเฉพาะ "เฟสที่ลูกค้าโฟกัสตอนนี้" (MVP)** — ลูกค้ายังอยู่ขั้นสำรวจ ห้ามรวมงานปลายทาง
  ที่ยังไม่ตัดสินใจ ให้เขียนไว้ใน assumptions/questions เพื่อให้ requirement เดียวกันได้ scope เท่ากันทุกครั้ง
- รองรับงานทุกประเภท ไม่ใช่แค่ซอฟต์แวร์ (research, ตั้งต้นธุรกิจ, ops) — ดูตารางเฟสใน SKILLS.md

## โดเมนที่พบบ่อย (ใช้เทียบเคียงการประเมิน)

- **E-Commerce / Retail**: สินค้า, สต็อกหลายช่องทาง, ออเดอร์ PO/SO, ชำระเงิน, รายงาน
- **Healthcare / Clinic**: จองคิว, โปรไฟล์คนไข้, แจ้งเตือน, ชำระเงิน
- **Loyalty**: สะสมแต้ม, สมาชิก, แลกของรางวัล, แจ้งเตือน
- **Logistics / WMS**: คลังสินค้า, barcode/QR, รับ-จ่ายของ, รายงานสต็อก
