ขั้นตอนที่ 1

1Clone โปรเจกต์

ดาวน์โหลดซอร์สโค้ดลงเครื่อง

terminal 
git clone https://github.com/Benz-srg/eazy-scan-irc.git
cd eazy-scan-irc
ต้องมีอย่างใดอย่างหนึ่ง: Docker Desktop (แบบ A) หรือ Node 20+ และ pnpm (แบบ B/C)
ขั้นตอนที่ 2

2รันโปรเจกต์

🅰️ แบบ A — Docker คำสั่งเดียว

ได้ web + Whisper + MongoDB ครบ (ครั้งแรกโหลด Whisper model ~3GB รอสักครู่)

terminal 
# 1) คัดลอกไฟล์ตั้งค่า แล้วใส่ API key 1 ตัวในไฟล์ .env
cp .env.example .env
# เปิด .env แล้วใส่ เช่น OPENAI_API_KEY=sk-...

# 2) รันทั้งโปรเจกต์
docker compose up --build
เปิด http://localhost:3000 — เสร็จ! · ใช้ key อื่นได้: ตั้ง LLM_PROVIDER=anthropic + ANTHROPIC_API_KEY (หรือ gemini) ใน .env

🅱️ แบบ B — Local ฟรี 100% (มี Claude Code CLI)

Claude CLI วิเคราะห์ + Local Whisper ถอดเสียง — ไม่ต้องมี API key

terminal 
pnpm install            # ติดตั้ง dependencies
pnpm whisper:setup      # ติดตั้ง Local Whisper (ครั้งแรกครั้งเดียว)
pnpm db:up              # MongoDB ผ่าน Docker (ข้ามได้ถ้าไม่ต้องการประวัติถาวร)
pnpm dev:all            # รัน web + Whisper พร้อมกัน

เชื่อม Claude Code CLI ก่อน (ครั้งเดียว):

terminal 
npm install -g @anthropic-ai/claude-code   # ติดตั้ง CLI
claude                                     # ล็อกอินผ่านเบราว์เซอร์
echo "hi" | claude -p --output-format json # เช็คว่าใช้ได้ (ต้องได้ JSON)
Auto-connect: ถ้าไม่ตั้งอะไรเลย ระบบใช้ Claude CLI ให้อัตโนมัติ · ถึงเผลอตั้ง provider เป็น OpenAI แต่ไม่มี key ก็ fallback มา CLI ให้

🅲 แบบ C — เดโม (0 setup)

terminal 
pnpm install && pnpm dev

เปิด localhost:3000 → กดปุ่ม "ดูตัวอย่าง" ที่หน้าแรก เห็นผลวิเคราะห์ตัวอย่างทันที

ขั้นตอนที่ 3

3ทดสอบการใช้งาน

ลองใช้งานจริงตามนี้ทีละขั้น

  1. เปิด http://localhost:3000 → กด "เริ่มใช้งานฟรี"
  2. ที่หน้า พื้นที่ทำงาน → อัดเสียง หรือ ลากไฟล์เสียง มาวาง (.mp3 .wav .m4a)
  3. เลือกบริการถอดเสียง (Local Whisper ฟรี หรือ OpenAI Whisper เร็วกว่า)
  4. กด "วิเคราะห์ Requirement" → ดูสเต็ปทำงานจริง พร้อมเวลาแต่ละขั้น
  5. เห็นผล: สรุปผู้บริหาร · ฟีเจอร์ (SOW) · ตาราง Manday · ความเสี่ยง · คำถามที่ควรถามลูกค้า
  6. กด ส่งออก เป็น DOCX / PDF · หรือ วิเคราะห์ใหม่ด้วย AI อื่น ถ้ายังไม่ถูกใจ
  7. ไปหน้า ประวัติการวิเคราะห์ → ค้นหา / เล่นเสียง / เปิดดูผลเก่า
ทดสอบผ่าน API (Postman): import postman/EazyScan.postman_collection.json + postman/EazyScan.postman_environment.json → มี request ครบ (Analyze แนบ samples/sample-requirement-th.m4a, Reanalyze ทดสอบ AI ตรง ๆ, Export, Projects)
ออกจากหน้าระหว่างประมวลผลได้ — งานรันต่อฝั่ง server แล้วโชว์ผลใน ประวัติ เมื่อกลับมา (ต้องเปิด MongoDB)
รายละเอียด

เครื่องมือวิเคราะห์ (LLM)

ตั้งผ่าน LLM_PROVIDER ใน .env — ทุกตัวใช้กฎ/คุณภาพชุดเดียวกัน

Providerต้องมีค่าใช้จ่าย
claude-cli (ดีฟอลต์ local)Claude Code CLI ล็อกอินฟรี (ไม่ต้องมี key)
anthropicANTHROPIC_API_KEYจ่ายตาม API
geminiGEMINI_API_KEYจ่ายตาม API
openai (ดีฟอลต์ Docker)OPENAI_API_KEYจ่ายตาม API

STT: Local Whisper (ฟรี, ในเครื่อง) หรือ OpenAI Whisper (เร็วกว่า, ใช้ key) — เลือกได้ในแอป

แก้ปัญหา

เจอปัญหาบ่อย ๆ

Docker daemon ไม่ขึ้น / compose ค้าง

เปิด Docker Desktop ให้ whale icon เขียวนิ่งก่อน (อาจมี dialog ให้กดยอมรับ). ถ้าค้าง → Quit แล้วเปิดใหม่

Local Whisper ช้า

large-v3 บน CPU ช้า (~60–210s/ไฟล์) แต่แม่นไทย. อยากเร็ว → เลือก OpenAI Whisper ในแอป (~15s)

วิเคราะห์ไม่ได้ / ขึ้น error

เช็คว่า claude -p ใช้ได้ (แบบ B) หรือใส่ API key ถูกต้อง (แบบ A). ถ้าไม่มีทั้งคู่ ระบบจะใช้ข้อมูลตัวอย่างแทน
